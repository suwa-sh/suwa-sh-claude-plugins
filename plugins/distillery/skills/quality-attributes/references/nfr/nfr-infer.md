# Step1: RDRA モデルからの NFR 推論

RDRA モデルを読み取り、非機能要求グレードを推論するタスク。

## 入力

- `docs/rdra/latest/*.tsv` — 現在の RDRA モデル（全ファイル）
- `docs/rdra/latest/システム概要.json` — システム概要
- `references/nfr-grade-catalog.md` — IPA 非機能要求グレードカタログ
- `references/nfr-inference-rules.md` — 推論ルール
- `references/nfr-grade-schema.md` — 出力スキーマ定義
- `docs/nfr/latest/nfr-grade.yaml`（差分更新モード時のみ）— 既存の NFR グレード

## タスク手順

### 1. RDRA モデルの読み込みと特徴抽出

以下の観点で RDRA モデルを分析する:

#### BUC.tsv から
- 業務の種別（24h稼働？バッチ？営業時間内？リアルタイム？）
- 業務の社会的影響度（公共性、金銭取引の有無）
- UC の数と複雑さ

#### アクター.tsv から
- アクター種別数
- 外部ユーザー（一般利用者、顧客等）の有無
- 管理者ロールの有無

#### 外部システム.tsv から
- 外部システム数
- 外部システムの種類（決済、認証、行政等）
- 連携方式の推定

#### 情報.tsv から
- 情報エンティティ数と属性数の合計
- 個人情報・機密情報の有無（氏名、住所、メール、電話、パスワード等）
- 金銭・取引関連の情報の有無

#### 状態.tsv から
- 状態遷移の複雑さ（状態数、遷移パス数）
- データ整合性が重要な状態の有無

#### 条件.tsv から
- アクセス制御・権限に関する条件の有無
- バリデーション条件の数

#### バリエーション.tsv から
- 選択肢の種類と数

#### システム概要.json から
- システムの目的・概要
- 対象ユーザー層

### 2. モデルシステムの推定

`nfr-inference-rules.md` の「モデルシステム推定ルール」に従い、model1/model2/model3 を推定する。

推定結果には以下を含める:
- 推定されたモデルシステム（model1/model2/model3）
- 推定根拠（どの RDRA 要素からどう判断したか）
- confidence（確信度）

### 3. カテゴリ別の推論

`nfr-inference-rules.md` の「カテゴリ別推論ルール」に従い、6大項目すべてのメトリクスについてグレードを推論する。

推論の優先順位:
1. **RDRA から直接推論できる項目**: confidence = high/medium で推論
2. **RDRA から間接的に推論できる項目**: confidence = low で推論
3. **推論不能な項目**: モデルシステムのデフォルト値を適用し、confidence = default

各メトリクスについて以下を決定する:
- grade: 推定レベル（0〜5）
- reason: 推定根拠
- source_model: 根拠となった RDRA モデル要素
- confidence: 確信度

### 4. 差分更新モード時の処理

既存の `docs/nfr/latest/nfr-grade.yaml` がある場合:

1. RDRA の差分（`docs/rdra/events/{最新event_id}/_changes.md`）を読み取る
2. 変更された RDRA 要素に関連する NFR 項目を特定する
3. 該当項目のみを再推論する
4. 既存グレードの `confidence: "user"` の項目は再推論しない（ユーザー確定値を尊重）
5. 再推論結果と既存値の差異をリストアップする

## 出力

このステップではファイル出力を行わない。推論結果は Step2（対話）に渡すための内部データとして保持する。

推論結果サマリを以下の形式で構成する:

```
## モデルシステム推定

推定: {model1/model2/model3}（confidence: {high/medium/low}）
根拠: {根拠説明}

## 推論結果サマリ

### 重要項目（confidence: high/medium）

| ID | メトリクス | 推定Lv | confidence | 根拠 |
|----|----------|--------|-----------|------|
| A.1.1.1 | 運用時間（通常） | 3 | high | BUC「〜」が営業時間外利用 |
| ... | ... | ... | ... | ... |

### 要確認項目（confidence: low/default）

| ID | メトリクス | 推定Lv | 確認理由 |
|----|----------|--------|---------|
| D.2.1.1 | 移行方式 | 1 | 既存システムの有無が不明 |
| ... | ... | ... | ... |

### 差分更新時: 変更項目（差分更新モードのみ）

| ID | メトリクス | 旧Lv | 新推定Lv | 変更理由 |
|----|----------|------|---------|---------|
| ... | ... | ... | ... | ... |
```

## 注意事項

- 推論は保守的に行う（迷ったら低めのグレードを推定し、Step2 で上げる方向で調整）
- 全238メトリクスについて必ずグレードを設定する（空欄は許可しない）
- RDRA モデルに記載のない情報を想像で補完しない
- 推論根拠は具体的な RDRA 要素名を引用する
