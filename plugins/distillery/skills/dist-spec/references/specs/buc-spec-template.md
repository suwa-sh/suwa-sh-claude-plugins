# BUC Spec テンプレート定義

> **読み込みタイミング**: Step3.5 で使用。BUC Spec フォーマット。

BUC（ビジネスユースケース）単位の俯瞰仕様。所属する UC を横断して、データフロー・状態遷移・共有される条件/バリエーションの全体像を提供する。UC Spec が全て出揃った後に生成する（Step3.5）。

## ディレクトリ構成

```
{業務名}/
  {BUC名}/
    buc-spec.md                         # BUC 俯瞰仕様
    {UC名}/
      spec.md                           # UC の仕様概要
      tier-{tier_id}.md                 # ティア別仕様
```

## buc-spec.md フォーマット

```markdown
# {BUC名}

## 概要

{BUC の目的と概要を1-3文で記述}

## 所属 UC 一覧

| UC名 | アクター | 主な操作 | 関連情報 |
|------|---------|---------|---------|
| [{UC名}]({UC名}/spec.md) | {アクター名} | {操作概要} | {主要な情報名} |

## UC 横断データフロー

BUC 内の UC 間で情報がどう流れるかを示す。情報がどの UC で作成（C）・参照（R）・更新（U）されるかを明記する。

### データフロー図

```mermaid
graph LR
  UC1["{UC名1}"] -->|"{情報名}"| UC2["{UC名2}"]
  UC2 -->|"{情報名}"| UC3["{UC名3}"]
```

### 情報 CRUD マトリクス

| 情報名 | {UC名1} | {UC名2} | {UC名3} | ... |
|--------|:-------:|:-------:|:-------:|:---:|
| {情報名} | C | R | U | |
| {情報名} | | C | R | |

## 状態遷移全体図

BUC 内で関連する状態モデルの全遷移パスと、各遷移を担当する UC を示す。

```mermaid
stateDiagram-v2
  [*] --> {初期状態}: {UC名}
  {初期状態} --> {状態A}: {UC名}
  {状態A} --> {状態B}: {UC名}
  {状態B} --> [*]
```

### 状態遷移 UC マッピング

| 状態モデル | 遷移元 | 遷移先 | 担当 UC |
|-----------|--------|--------|--------|
| {状態モデル名} | {遷移元} | {遷移先} | [{UC名}]({UC名}/spec.md) |

## BUC 内共有条件一覧

BUC 内の複数 UC で共有される条件の一覧。各条件がどの UC で適用されるかを明記する。

| 条件名 | 条件の説明 | 適用 UC |
|--------|----------|--------|
| {条件名} | {条件の説明} | {UC名1}, {UC名2} |

## BUC 内共有バリエーション一覧

BUC 内の複数 UC で共有されるバリエーションの一覧。

| バリエーション名 | 値 | 適用 UC |
|----------------|---|--------|
| {バリエーション名} | {値} | {UC名1}, {UC名2} |
```

## 生成ルール

- BUC.tsv から BUC 内の全 UC を特定する
- 各 UC の spec.md の RDRA トレーサビリティテーブル（分岐条件一覧、バリエーション一覧、状態遷移一覧）を集約する
- 情報.tsv から BUC 内で扱われる情報を特定し、CRUD マトリクスを生成する
- 状態.tsv から BUC 内の状態遷移を全て抽出し、stateDiagram-v2 で描画する
- 条件・バリエーションは、2つ以上の UC で共有されるもののみ「共有」として記載する（1つの UC でしか使われないものは UC Spec 側のみ）
