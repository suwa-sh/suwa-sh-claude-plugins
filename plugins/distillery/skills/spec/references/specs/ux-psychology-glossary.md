# UX 心理学用語集

UX デザインに活用する心理学効果・バイアス・設計原則の定義。Spec 生成時に `_cross-cutting/ux-ui/ux-design.md` のインタラクション設計原則で適用する原則を選定する際に参照する。

## 認知・知覚系

| 原則 | 説明 | 設計への適用 |
|------|------|-----------|
| **認知負荷 (Cognitive Load)** | 情報処理に消費する精神的エネルギー量 | 一画面の情報量を4-5項目に制限。段階的開示で必要な時に表示 |
| **バナー・ブラインドネス (Banner Blindness)** | 広告的要素を無意識に無視する現象 | 重要な通知を広告的デザインにしない。インライン表示を推奨 |
| **選択的注意 (Selective Attention)** | 重要情報だけに注意を向け他を無視する機能 | コントラスト・サイズで注意を誘導。視覚的階層を明確にする |
| **視覚的階層 (Visual Hierarchy)** | 大きさ・色・配置で情報の重要度を整理する手法 | 見出し > 本文 > 補足の階層。アクションボタンの強調 |

## 判断・意思決定系

| 原則 | 説明 | 設計への適用 |
|------|------|-----------|
| **アンカー効果 (Anchor Effect)** | 最初に提示された情報が基準になる | 価格表示で定価→割引価格の順。比較表で最初に推奨プラン |
| **確証バイアス (Confirmation Bias)** | 自分の信念に合う情報を優先する傾向 | 検索結果でバランスの取れた情報提示 |
| **決断疲れ (Decision Fatigue)** | 繰り返しの決断で合理的選択が困難になる | 選択肢は3-5個に制限。デフォルト値を設定 |
| **おとり効果 (Decoy Effect)** | 魅力の低い選択肢で特定プランを魅力的に見せる | 料金プランの中間プランを強調 |
| **デフォルト効果 (Default Bias)** | 初期設定のまま選択し続ける傾向 | 推奨設定をデフォルトに。オプトイン/オプトアウトの設計 |
| **フレーミング効果 (Framing)** | 提示方法によって判断が左右される | 「95%成功率」vs「5%失敗率」の使い分け |

## 動機づけ・行動系

| 原則 | 説明 | 設計への適用 |
|------|------|-----------|
| **好奇心ギャップ (Curiosity Gap)** | 情報の欠如を埋めたい欲求 | 「あと3ステップで完了」のプログレス表示 |
| **ゲーミフィケーション (Gamification)** | ゲーム要素でモチベーション向上 | バッジ、ポイント、リーダーボード |
| **目標勾配効果 (Goal Gradient Effect)** | ゴールが近づくと行動が加速する | プログレスバーで完了率を可視化 |
| **損失回避 (Loss Aversion)** | 損失を避けることを優先する心理 | 「この特典は今日まで」の期限表示 |
| **ナッジ効果 (Nudge)** | 自発的な行動変容を促す介入 | 推奨オプションの視覚的強調 |
| **希少性効果 (Scarcity)** | 限定性に高い価値を感じる | 「残り3席」の在庫表示 |
| **社会的証明 (Social Proof)** | 他者の行動を信頼して同調する | レビュー数、利用者数の表示 |
| **変動型報酬 (Variable Reward)** | ランダムな報酬で行動を継続させる | フィード更新、通知のランダム性 |

## 体験評価系

| 原則 | 説明 | 設計への適用 |
|------|------|-----------|
| **美的ユーザビリティ効果 (Aesthetic-Usability)** | 美しいデザインは使いやすく感じる | 一貫したデザインシステムの適用 |
| **ドハティの閾値 (Doherty Threshold)** | 0.4秒超でユーザーの興味喪失リスク | レスポンスタイム0.4秒以内。ローディング表示 |
| **ピーク・エンドの法則 (Peak-End Rule)** | 体験評価はピークと最後で決まる | 完了画面の達成感演出。エラー時の丁寧な回復 |
| **ユーザー歓喜効果 (User Delight)** | 予期せぬ喜びや驚きを与える体験 | マイクロインタラクション。成功時のアニメーション |

## 所有・継続系

| 原則 | 説明 | 設計への適用 |
|------|------|-----------|
| **授かり効果 (Endowment Effect)** | 所有しているものを高く評価する | 無料トライアル。カスタマイズ後のデータ蓄積 |
| **サンクコスト効果 (Sunk Cost)** | 既投資を取り戻そうとする心理 | プロフィール完成度表示。使い続ける動機付け |
| **ツァイガルニク効果 (Zeigarnik Effect)** | 未完了タスクが強く記憶に残る | 未完了のオンボーディングステップの表示 |

## UX デザインパターン系

| 原則 | 説明 | 設計への適用 |
|------|------|-----------|
| **段階的開示 (Progressive Disclosure)** | 必要に応じて段階的に情報表示 | 「詳細を表示」の折りたたみ。ウィザード形式 |
| **段階的要請 (Foot in the Door)** | 小さなお願いから大きなお願いへ | 会員登録前に名前だけ入力 → 詳細入力 |
| **意図的な壁 (Intentional Friction)** | 誤操作防止のための遅延 | 削除前の確認ダイアログ。送金時の二段階確認 |
| **労働の錯覚 (Labor Illusion)** | 処理に労力がかかるように見せる | 検索結果のプログレス演出（即答でも少し待たせる） |
| **反応型オンボーディング (Reactive Onboarding)** | 行動に応じたガイド提示 | コンテキストに応じたツールチップ |
| **プライミング効果 (Priming)** | 先行刺激が後続判断に影響 | ランディングページのビジュアルで印象形成 |
| **ビジュアル・アンカー (Visual Anchor)** | 視覚的に目立つ要素で注意を引く | CTA ボタンの色・サイズによる誘導 |
| **系列位置効果 (Serial Position Effect)** | リスト先頭と末尾が記憶に残る | ナビゲーションの先頭と末尾に重要項目を配置 |

## Spec での活用方法

1. `_cross-cutting/ux-ui/ux-design.md` の「UX 心理学に基づくインタラクション設計原則」で、このシステムに適用する原則を選定しテーブルに記載する
2. 選定基準: システムの性質（BtoB/BtoC、業務系/一般向け等）と RDRA のアクター特性に合わせて選ぶ
3. 各原則について「適用場面」と「具体的な設計」を記述する
4. 過度な適用は認知負荷を上げるため、重要度の高い原則を 5-10 個程度選定する
