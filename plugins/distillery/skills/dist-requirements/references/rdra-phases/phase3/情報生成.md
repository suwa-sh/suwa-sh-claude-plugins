# phase3：情報
- 「0_RDRAZeroOne/phase1/初期要望分析.md」を読み初期要望を理解する
- 「0_RDRAZeroOne/phase2/情報.tsv」を読み情報を理解する
- 「0_RDRAZeroOne/phase2/状態.tsv」を読み状態モデルを理解する
- 「0_RDRAZeroOne/phase1/バリエーション.tsv」を読みバリエーションを理解する
- 上記ファイルから情報を再整理し「情報.tsv」を出力する

# 背景理解
- ここではRDRAモデルの中の情報を整備する

## 情報の整理
- 「phase2/情報.tsv」をベースにコンテキストで分類し、状態モデル、バリエーションと紐付けて「phase3/情報.tsv」に出力する
- 情報を役割に応じてコンテキストとして分類する
- コンテキストは情報を中心とした管理単位となる
- コンテキストはドメインのグループとして活用する
- 「phase2/情報.tsv」を順次読込以下のことを行う
  - 情報に関係する状態モデルを関連付ける
  - 情報はユニークなIDを持ち、状態モデルもユニークなIDの状態遷移を表す
  - 同じIDをもつものを関連付ける
  - 情報に従属するバリエーションを関連付ける
    - 例：会員ランク（バリエーション）は会員（情報）に従属する
  - 初期要望分析にUCと情報、条件（ビジネスルール）、外部システムの関係の記述があれば、それを含める
  - 情報のIDと同じIDをもつ「状態.tsv」の状態モデルは、状態モデルカラムに設定する
  - バリエーションが情報に従属する場合はバリエーションに設定する　例:(バリエーション/会員ランクは会員情報に従属する)
- 上記の操作を行い「情報」のフォーマットで出力する

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase3/情報.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する

### 「情報」のフォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: コンテキスト：情報を役割に応じてグルーピングする。名前の後ろに「～管理」を付ける
2列目: 情報：ユニークなIDで一意に識別できるデータを表す
3列目: 属性：情報が保持する複数の属性を表す。属性は「、」区切りで一つの文字列として保持する
4列目: 関連情報：情報同士の関連を表す。情報と関連する情報を「、」区切りで一つの文字列として保持する
5列目: 状態モデル：状態で管理する状態モデルを表します、「、」区切りで複数の状態モデルを一つの文字列で表す
6列目: バリエーション：情報に従属するバリエーションを表します、「、」区切りで複数のバリエーションを一つの文字列で表す
7列目: 説明：情報の内容、活用法の説明

ヘッダー行を含め、各フィールドはタブ文字で区切ってください。

## 出力チェックリスト
- [ ] 5列目:状態モデルカラムに設定する名前は「状態.tsv」に定義されている状態モデルを設定、新たに状態モデルを生成しない
- [ ] 6列目:バリエーションカラムは「バリエーション.tsv」に定義されているバリエーションを設定、新たにバリエーションを生成しない
