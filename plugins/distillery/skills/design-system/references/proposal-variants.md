# デザインシステム確認項目 — 3案雛形（proposal-variants）

design-system スキルがユーザー確認時に提示する 3案の雛形。
全て `skills/pipeline/references/dialogue-format.md` に準拠する
（3案以上 + ⭐推奨 + 一行説明 + 推奨理由）。

## 1. ブランド方針

- **Option A** (⭐推奨): 信頼・堅実路線 — BtoB / 業務系に向く。保守的だが幅広い層に安心感
- **Option B**: 革新・先進路線 — スタートアップ / 新規事業向け。差別化強いが保守層に刺さらない可能性
- **Option C**: 親近・カジュアル路線 — BtoC / 個人ユーザー向け。親しみやすいが業務系には軽く見える

**推奨理由**: RDRA の主要アクターと BUC のトーンから推論する。

## 2. カラーパレット（プライマリ系統）

- **Option A** (⭐推奨): 信頼感ブルー系 — `#1E40AF` / `#3B82F6` / `#DBEAFE`。金融・業務系の定番、可読性が高い
- **Option B**: 落ち着きグリーン系 — `#065F46` / `#10B981` / `#D1FAE5`。医療・教育・サステナビリティ系に適合
- **Option C**: ブランドパープル系 — `#5B21B6` / `#8B5CF6` / `#EDE9FE`。差別化が効き、クリエイティブ系に向く

**推奨理由**: ブランド方針の Option と業種から推論。各 Option はプライマリ / アクセント / 背景の 3 色で構成。

## 3. タイポグラフィ（和欧ペア）

- **Option A** (⭐推奨): Noto Sans JP + Inter — モダンで可読性最高、無償、幅広い用途に適合
- **Option B**: Noto Serif JP + Source Serif 4 — 信頼感と格調、長文向き、やや堅い印象
- **Option C**: M PLUS 1p + Work Sans — 親近感と丸みのある印象、カジュアル寄り

**推奨理由**: ブランド方針と読み物量（spec の UC 数）から推論。

## 4. レイアウト方針

- **Option A** (⭐推奨): 12 列グリッド + サイドバーナビ — 業務系 / 管理画面の定番。情報密度が高い
- **Option B**: シングルカラム中心 + トップナビ — BtoC / ランディング向き。読みやすいが密度は低い
- **Option C**: カード主体 + ヘッダーナビ — ダッシュボード / ポータル向き。視認性は高いが縦に伸びやすい

**推奨理由**: RDRA の画面定義と BUC 数から推論。

## 5. コンポーネントスタイル（角丸 / 影）

- **Option A** (⭐推奨): 中間（radius 8px, shadow sm）— 汎用、モダンで使いやすい
- **Option B**: シャープ（radius 2px, shadow none）— ミニマル / プロフェッショナル
- **Option C**: 柔らかい（radius 16px, shadow md）— フレンドリー / BtoC 寄り

**推奨理由**: ブランド方針との整合性。

## 6. アイコノグラフィ

- **Option A** (⭐推奨): Lucide — オープンソース、SVG、軽量、型付き React ラッパーあり
- **Option B**: Heroicons — Tailwind 製、outline/solid 2 系統、React 公式
- **Option C**: Material Symbols — Google 製、3 段階ウェイト、アクセシビリティ対応

**推奨理由**: 既存コンポーネントスタック（Tailwind / shadcn 等）との親和性。

---

## 使用ルール

- **必ず Option は 3 案以上** 提示する
- **必ず 1 つに `(⭐推奨)` マーク** を付ける
- **各 Option に一行説明**（選択理由・効果・代償）を付ける
- **推奨理由には根拠**（どの RDRA / NFR / ブランド方針から推論したか）を明記
- 画面追加（RDRA 未定義）は **自動追加禁止**。`appendTodo.js` で `docs/todo.md` に提案を登録し、
  確認推奨項目として返却する
