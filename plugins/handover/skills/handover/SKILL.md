# セッション引き継ぎドキュメント生成

現在の会話コンテキストから、セッション引き継ぎドキュメントを生成してください。

## 手順

1. 現在の会話コンテキストを振り返り、以下のテンプレートに沿って HANDOVER.md を作成
2. cwd（カレントディレクトリ）に `HANDOVER.md` を Write ツールで書き出す
3. 完了を報告

## 追加の指示

$ARGUMENTS

## HANDOVER.md テンプレート

以下のフォーマットで生成すること。各セクションは箇条書きで簡潔に記載する。

```markdown
# セッション引き継ぎ

## 取り組みと完了状況
- （何に取り組み、何が完了したか）

## 成功と課題
- （うまくいったこと、いかなかったこと。バグや修正を含む）

## 主要な意思決定と理由
- （重要な判断とその根拠）

## 教訓と注意点
- （学んだこと、ハマりポイント）

## ネクストステップ
- （次のアクション、優先度順）

## 重要ファイルマップ
- （重要ファイルの絶対パスと役割）
```

## 注意事項

- 簡潔さを重視する。各項目は1-2行で十分
- コードの詳細よりも「何をしたか」「なぜそうしたか」に焦点を当てる
- ネクストステップは具体的かつ実行可能な形で記載する
- 重要ファイルマップには変更・作成したファイルと、参照が必要なファイルを含める
