## 状態モデル → ステータス表示

### 書籍貸出状態 (3状態 + 終了)
- 在庫あり → green（利用可能）
- 貸出中 → blue（処理中）
- 延滞中 → red（問題あり）

### 予約状態 (3状態 + 終了)
- 予約受付中 → blue（処理中）
- 予約確保済 → green（利用可能）
- 予約キャンセル → gray（無効）

## NFR → デザイン判断

| NFR | グレード | デザイン判断 |
|-----|---------|-------------|
| A.1.1.1 運用時間 | Lv2 (9時-21時) | ローディング/エラー状態を全画面に配置 |
| B.1.1.1 同時アクセス | Lv1 (~100) | ページネーション必要（仮想スクロール不要） |
| B.2.1.1 レスポンスタイム | Lv2 (10秒以内) | Skeleton UI でパーシーブドパフォーマンス改善 |
| E.5.1.1 認証方式 | Lv2 (OAuth2/OIDC) | マネージド IdP ログイン画面（カスタムUI不要） |
| E.5.2.1 認可方式 | Lv2 (RBAC) | ポータル別ナビゲーション・権限ベース表示切替 |
| E.1.2.1 PII | 個人情報あり | 利用者情報にマスク表示考慮 |

## Arch → 技術判断

| Arch 要素 | デザイン判断 |
|-----------|-------------|
| TypeScript + Next.js | React コンポーネント、SSR 対応 |
| SSR (tier-frontend) | 蔵書検索画面は SSR で初回表示速度確保 |
| 2ポータル (利用者/司書) | data-portal 属性によるテーマ切替 |
| OAuth2/OIDC (CTP-001) | ログインは IdP Hosted UI に委譲 |
| 日本語のみ (CTP-007) | i18n 不要、テキスト直接記述 |
| FaaS + REST API | API レスポンスに基づくローディング/エラー表示 |
