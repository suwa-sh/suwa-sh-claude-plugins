# USDM 要求仕様書

- システム名: 図書館蔵書管理システム
- イベントID: 20260412_140535_initial_build
- 作成日時: 2026-04-12T14:05:35
- ソース: 初期要望.txt

| ID | 要求 | 仕様 | 理由（背景） | 説明 |
|----|------|------|------------|------|
| REQ-001 | 図書館の蔵書と利用者を一元管理できるシステムを構築する | | 蔵書と利用者の情報が分散しており、一元的に管理することで業務効率と利用者サービスを向上させるため | |
| SPEC-001-01 | | 書籍情報（タイトル、著者、ISBN、出版社、ジャンル等）を登録・編集・削除できる | | 1. Given 司書がログイン済み When 書籍登録画面でISBN・タイトル・著者を入力して登録する Then 書籍が蔵書として登録され一覧に表示される<br>2. Given 登録済み書籍が存在する When 司書が書籍情報を編集する Then 変更内容が反映される |
| SPEC-001-02 | | 利用者情報（氏名、連絡先、利用者番号等）を登録・編集・削除できる | | 1. Given 司書がログイン済み When 利用者登録画面で氏名・連絡先を入力して登録する Then 利用者が登録され利用者番号が発行される |
| SPEC-001-03 | | 書籍をキーワード、タイトル、著者、ISBN、ジャンルで検索できる | | 1. Given 利用者がシステムにアクセス済み When 検索条件を入力して検索する Then 条件に合致する書籍の一覧が表示される |
| REQ-002 | 書籍の貸出・返却・予約をWeb画面から行えるようにする | | 貸出・返却・予約業務をシステム化することで、手作業による管理の手間やミスを削減し、利用者の利便性を高めるため | |
| SPEC-002-01 | | 利用者が書籍の貸出手続きをWeb画面から行える | | 1. Given 利用者が貸出可能な書籍を選択済み When 貸出ボタンを押す Then 貸出が記録され返却期限が設定される<br>2. Given 書籍が貸出中の場合 When 別の利用者が貸出を試みる Then 貸出不可のメッセージが表示される |
| SPEC-002-02 | | 利用者が書籍の返却手続きをWeb画面から行える | | 1. Given 利用者が貸出中の書籍を選択済み When 返却ボタンを押す Then 返却が記録され書籍が貸出可能になる |
| SPEC-002-03 | | 利用者が貸出中の書籍に対して予約を行える | | 1. Given 書籍が貸出中 When 利用者が予約ボタンを押す Then 予約が記録され予約順が設定される<br>2. Given 予約済み書籍が返却された When 予約順1位の利用者に通知する Then 利用者に予約可能通知が届く |
| REQ-003 | 貸出期限の管理や延滞者への通知を自動化する | | 貸出期限の管理と延滞通知を手動で行うと漏れが発生しやすく、自動化により確実な運用と司書の負荷軽減を実現するため | |
| SPEC-003-01 | | 貸出時に返却期限を自動設定し、期限が近づいたら利用者にリマインド通知を送信する | | 1. Given 貸出が記録済み When 返却期限3日前になる Then 利用者にリマインドメールが送信される |
| SPEC-003-02 | | 返却期限を超過した場合に延滞者へ督促通知を自動送信する | | 1. Given 返却期限を超過した貸出が存在する When 日次バッチが実行される Then 延滞者にメールで督促通知が送信される |
| REQ-004 | 利用者が自分の貸出履歴や予約状況を確認できるようにする | | 利用者自身が貸出状況や予約状況をいつでも確認できることで、自己管理が可能になり問い合わせ対応の負荷も軽減されるため | |
| SPEC-004-01 | | 利用者が自分の貸出履歴（過去・現在）をWeb画面で確認できる | | 1. Given 利用者がログイン済み When マイページの貸出履歴画面を開く Then 過去と現在の貸出一覧が表示される |
| SPEC-004-02 | | 利用者が自分の予約状況をWeb画面で確認できる | | 1. Given 利用者がログイン済み When マイページの予約状況画面を開く Then 現在の予約一覧と予約順が表示される |
| REQ-005 | 司書が在庫状況や人気書籍の統計を把握できるようにする | | 在庫状況や貸出傾向を可視化することで、蔵書の補充計画や運営改善に役立てるため | |
| SPEC-005-01 | | 司書が書籍の在庫状況（貸出中・在庫あり・予約待ち）を一覧で確認できる | | 1. Given 司書がログイン済み When 在庫状況画面を開く Then 全書籍の在庫状態が一覧表示される |
| SPEC-005-02 | | 司書が人気書籍ランキングや貸出統計をレポートとして確認できる | | 1. Given 司書がログイン済み When 統計レポート画面を開く Then 貸出回数ランキング・期間別統計が表示される |
| REQ-006 | 将来的に電子書籍にも対応できるようにする | | 電子書籍への対応を見据えた設計にしておくことで、将来の拡張時に大規模な改修を避けるため | |
| SPEC-006-01 | | 書籍情報に資料種別（紙書籍・電子書籍）を持たせ、将来の電子書籍対応に備える | | 1. Given 書籍登録画面 When 資料種別を選択できる Then 紙書籍として登録でき、将来電子書籍も選択可能な設計となっている |
