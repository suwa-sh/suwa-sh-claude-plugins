# 書籍を削除する - フロントエンド仕様

## 変更概要

蔵書管理画面に削除機能を追加する。確認ダイアログによる誤操作防止を実装する。

## 画面仕様

### 蔵書管理画面（削除機能）

- **URL**: /admin/books
- **アクセス権**: 司書ロール
- **ポータル**: admin

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 書籍一覧 | テーブル | BookCard, BookLoanStatusBadge | 書籍一覧表示 |
| 検索フィルター | フォーム | BookSearchFilter | キーワード・ジャンル・資料種別フィルター |
| 削除ボタン | ボタン | Button (destructive, size: sm) | 在庫あり状態の書籍のみ有効 |
| 確認ダイアログ | モーダル | Card + Button (destructive) + Button (outline) | 「この書籍を削除しますか？」 |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 削除ボタン色 | var(--semantic-destructive) | #DC2626 |
| ダイアログ背景 | var(--component-card-bg) | #FFFFFF |

#### UIロジック

- **状態管理**: 削除対象の bookId を state で管理。ダイアログの開閉状態
- **バリデーション**: 削除ボタンは在庫あり（available）状態の書籍のみ有効（貸出中/延滞中は disabled）
- **ローディング**: 削除実行中はダイアログの「削除する」ボタンを disabled + Spinner
- **エラーハンドリング**: 409 Conflict（貸出中）の場合はエラーバナー表示

#### 操作フロー

1. 蔵書管理画面で書籍の削除ボタン（赤色）をクリック
2. 確認ダイアログ表示「書籍『{タイトル}』を削除しますか？この操作は取り消せません」
3. 「削除する」クリック → DELETE /api/v1/books/:id 送信
4. 成功（204）→ ダイアログ閉じる → 一覧再取得
5. 「キャンセル」クリック → ダイアログ閉じる
