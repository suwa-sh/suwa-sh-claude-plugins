# 在庫状況を確認する - フロントエンド仕様

## 変更概要

司書ポータルに在庫状況画面を実装する。

## 画面仕様

### 在庫状況画面

- **URL**: (design-event.yaml screens参照)
- **アクセス権**: 司書ロール
- **ポータル**: admin

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| メイン表示 | 表示 | Card, Badge | 主要情報の表示 |
| アクションボタン | ボタン | Button (default) | 主要操作 |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| アクセント | var(--portal-primary) | #334155 |

#### UIロジック

- **状態管理**: React useState でフォーム/一覧 state を管理
- **バリデーション**: 必須チェック、型チェック
- **ローディング**: データ取得時は Skeleton UI
- **エラーハンドリング**: API エラー時はエラーバナー（RFC 7807 detail 表示）

#### 操作フロー

1. 画面にアクセス
2. データの表示/入力
3. アクション実行
4. 結果表示
