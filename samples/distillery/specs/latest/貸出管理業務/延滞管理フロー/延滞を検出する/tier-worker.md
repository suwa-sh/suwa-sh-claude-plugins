# 延滞を検出する - バックエンドワーカー仕様

## 変更概要

延滞を検出するの非同期処理/バッチ処理を実装する。

## イベント処理仕様

### 延滞を検出するハンドラ

- **トリガー**: 日次バッチ（CronJob: 毎日 22:00 JST）
- **入力チャネル**: (CronJobトリガー)
- **出力チャネル**: (なし)

#### 処理フロー

1. トリガーにより処理開始
2. 対象データの取得
3. ビジネスルール適用
4. 結果の永続化/通知送信
5. 処理完了ログ記録

#### エラーハンドリング

| エラー種別 | リトライ | DLQ | 説明 |
|-----------|---------|-----|------|
| 一時的エラー（DB接続） | Yes（3回） | No | 指数バックオフでリトライ |
| 永続的エラー（データ不整合） | No | Yes | DLQ に送信して手動対応 |
| 外部サービスエラー | Yes（3回） | Yes | リトライ後も失敗の場合は DLQ |

## データモデル変更

関連テーブル: 貸出
