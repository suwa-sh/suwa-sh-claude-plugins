# 書籍を削除する - バックエンドAPI仕様

## 変更概要

書籍削除 API を実装する。在庫あり状態の書籍のみ削除を許可する。

## API 仕様

### 書籍削除 API

- **メソッド**: DELETE
- **パス**: /api/v1/books/{id}
- **認証**: OAuth2/OIDC (司書ロール必須)

#### エラーレスポンス

| ステータスコード | 条件 | レスポンス |
|----------------|------|-----------|
| 404 | 書籍が存在しない | RFC 7807: detail="書籍が見つかりません" |
| 409 | 貸出中/延滞中の書籍 | RFC 7807: detail="貸出中の書籍は削除できません" |

## データモデル変更

### books テーブル

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| (全カラム) | - | 該当レコードを削除 | 削除 |

## ビジネスルール

- 書籍貸出状態が「在庫あり」の場合のみ削除可能
- 貸出中・延滞中の書籍は 409 Conflict
- 削除時に関連する予約（予約受付中）があればキャンセル処理を連鎖実行

## ティア完了条件（BDD）

```gherkin
Feature: 書籍を削除する - バックエンドAPI

  Scenario: 在庫あり書籍の削除成功
    Given 「在庫あり」状態の書籍 id="abc-123" が存在する
    When DELETE /api/v1/books/abc-123 を送信する
    Then HTTP 204 が返却される
    And books テーブルに id="abc-123" のレコードが存在しない

  Scenario: 貸出中書籍の削除拒否
    Given 「貸出中」状態の書籍 id="def-456" が存在する
    When DELETE /api/v1/books/def-456 を送信する
    Then HTTP 409 が返却される
    And books テーブルに id="def-456" のレコードが存在する
```
